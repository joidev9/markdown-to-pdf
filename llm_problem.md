## 1. Hallucination (환각)
모르는 걸 아는 척 그럴듯하게 답변하는 현상.
- 존재하지 않는 API, 함수, 패키지를 자신 있게 제안
- 구버전 정보를 최신인 것처럼 설명

## 2. Context Window Overflow (컨텍스트 초과)
대화가 길어지면 앞부분을 잊어버리는 현상.
- 초반에 합의한 내용을 후반에 무시
- 파일을 읽었는데 나중에 내용을 잊고 엉뚱하게 수정

## 3. Yolo Mode / Overconfidence (과잉 자신감)
확인 없이 바로 실행해버리는 문제.
- 가상환경 확인 없이 전역 설치
- 기존 코드를 확인 안 하고 덮어쓰기

## 4. Scope Creep (범위 확장)
요청하지 않은 것까지 과하게 손대는 현상.
- 버그 하나 고쳐달랬는데 리팩토링까지 진행
- 주변 코드에 타입 힌트, 독스트링 추가

## 5. Sycophancy (아첨/동조)
사용자 말에 무조건 동의하는 경향.
- 잘못된 접근을 제안해도 "좋은 생각입니다"라고 동조
- 반대 의견을 내야 할 때도 그냥 따라감

## 6. Regression (회귀)
하나를 고치면서 다른 걸 망가뜨리는 현상.
- A 파일 수정하면서 B 파일의 의존성을 깨뜨림
- 이전에 고쳤던 버그를 다시 만들어냄

## 7. Lazy Output (게으른 출력)
코드 일부를 `// ... 나머지 동일` 같은 주석으로 생략하는 현상.
- 파일이 길면 중간을 생략하고 앞뒤만 출력
- 복붙하면 코드가 누락됨

## 8. Anchor Bias (앵커 편향)
처음 선택한 잘못된 접근에 계속 집착하는 현상.
- 안 되는 방법을 미세하게 변형하며 반복 시도
- 다른 접근을 고려하지 않고 같은 삽질을 반복
